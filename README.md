# Resumo: Computação em Nuvem - Microsoft Azure (AZ-900)

## Visão Geral do Lab

Este lab apresenta uma introdução prática ao portal Microsoft Azure, focado na preparação para a certificação **AZ-900 (Azure Fundamentals)**. O principal objetivo é ensinar como localizar e organizar serviços no portal Azure por categoria.

## Principais Conceitos Abordados

### 1. Criando uma Conta Gratuita no Azure
- Acesso ao portal do Azure requer uma conta Microsoft
- A conta gratuita oferece acesso aos mesmos serviços que contas empresariais
- Algumas funcionalidades podem ter restrições por custo ou disponibilidade regional

### 2. Personalização do Portal Azure
- **Mudança de Idioma**: Opção disponível nas configurações
- **Modo de Exibição**: Alterar tema (claro/escuro) e layout dos menus
- **Interface Intuitiva**: O portal é user-friendly mesmo para iniciantes

### 3. Organização de Serviços por Categoria

Os serviços do Azure são organizados em categorias como:

- **Computação**: Máquinas Virtuais, App Services, Containers
- **Inteligência Artificial & Machine Learning**: IA, ML Services
- **Banco de Dados**: SQL Database, Cosmos DB, MySQL, PostgreSQL
- **DevOps**: Azure DevOps, GitHub Integration
- **Redes**: Virtual Networks, Load Balancer, VPN Gateway, Bastion/Jump Server
- **Armazenamento**: Blob Storage, File Share, Queue Storage
- **Migração**: Database Migration Service, Data Factory

### 4. Recursos em Versão Prévia (Preview)

- Identificáveis pela etiqueta "Preview" nos serviços
- **Não possui SLA (Service Level Agreement)** – não há garantia de funcionamento estável
- Serços em Preview não devem ser usados em produção crítica
- Ideais para testes e experimentação

### 5. Exemplos Práticos de Serviços

#### Computação
- **Máquinas Virtuais**: Servidores escalabilidade na nuvem
- **App Services**: Hospedagem de aplicações web

#### Redes e Segurança
- **Virtual Networks**: Criação de ambientes isolados
- **Bastion/Jump Server**: Acesso seguro a recursos privados
- **Security Groups**: Controle de tráfego de rede

#### Armazenamento e Dados
- **Blob Storage**: Armazenamento de dados não estruturados
- **Migração de Dados**: Tools para mover dados para a nuvem

## Insights Importantes

### Computação em Nuvem é Essencial em TI

- Praticamente todas as áreas de TI requerem conhecimento de nuvem:
  - Desenvolvimento de Aplicações
  - Administração de Banco de Dados
  - Arquitetura de Sistemas
  - DevOps e CI/CD
  - Inteligência Artificial e Machine Learning

### Por que Azure?

- Empresa Microsoft: Suporte robusto e inovaração constante
- Integração com ferramentas Microsoft (Office 365, Teams, Dynamics 365)
- Ampla gama de serviços em todas as áreas de TI
- Presença global com data centers em múltiplas regiões

## Conclusão

Este lab proporciona uma base sólida para entender como os serviços do Azure são organizados e acessados. A compreensão dessa estrutura é fundamental para:

- Passar no exame AZ-900
- Trabalhar eficientemente com Azure em projetos reais
- Tomar decisões arquiteturais informadas
- Administrar e otimizar recursos em nuvem
